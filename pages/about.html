Project Overview

This toolkit provides interactive simulations and visualizations to compare two common retail strategies:

  • Dollar Cost Averaging (DCA) — periodic fixed-dollar investments (default: weekly). The frontend simulates weekly purchases, aggregates shares purchased and money invested, and plots cumulative invested vs portfolio value for the selected timeframe.
  • Buy‑on‑Dip (BOD) — night‑before limit orders placed at previous close minus N% (configurable 1%–10%). A level is marked filled for a day when that day’s Low ≤ limit price. The frontend can use the precomputed data/all_buy_on_dip.csv fast‑path or recompute fills from data/history_tickers.csv.

How calculations are produced

Calculations are implemented by a two-step flow:

  1. ETL (Python) — etl-market-data.py pulls OHLCV daily history (Yahoo via yfinance), normalizes rows and writes data/history_tickers.csv. When configured, the ETL also writes data/all_buy_on_dip.csv containing normalized BOD execution events.
  2. Frontend (JS) — pages load CSVs and perform timeframe filtering. All period‑filtered cumulatives (shares, invested, value) are recomputed on the client from per‑row fields so the charts and tooltips reflect only the user selected date range.

Data files used by the site

  • data/history_tickers.csv — per‑day OHLCV with weekday and helper fields.
  • data/all_buy_on_dip.csv — precomputed BOD events (limit price, executed level/price, shares, dollars invested). The advanced BOD page uses this file when present for faster calculations.

Open, High, Low, Close, Volume — what each field means

Below are plain-language definitions for the daily fields that appear in the CSVs and on the site (these are not abbreviated here):

  • Open — the price of the first trade recorded for the symbol on that trading day.
  • High — the highest trade price reached for the symbol during that trading day.
  • Low — the lowest trade price reached for the symbol during that trading day.
  • Close — the price of the last trade at market close for that trading day. (Some datasets also include an Adjusted Close that corrects historical prices for splits and dividends; this site uses the unadjusted Close for day-by-day calculations unless an adjusted field is present and noted.)
  • Volume — the total number of shares traded for the symbol during that trading day (raw share count).

How the site uses these fields: the Dollar Cost Averaging tools use an average daily price (derived from the daily prices) for simulated purchases and use the Close price to compute portfolio value and percent gain; the Buy-on-Dip tools compare the previous day's Close to the day's Low to determine whether a limit order at a percentage below the previous Close would have executed.

User guidance

  • Default page state: ALL tickers + YTD.
  • Period and ticker choices are independent after load — selecting a ticker does not reset the period and vice‑versa.
  • Tooltips and metrics are calculated from rows in the chosen timeframe. Cumulative portfolio value = sum(shares) × close price on each date; cumulative invested = running sum of 'Dollars Invested'.

Advanced BOD notes

The advanced BOD tool lets users map share quantities to decline thresholds (1%–10%). When data/all_buy_on_dip.csv exists the page filters those precomputed events by ticker, date range, and decline-level to build the running totals and chart.
